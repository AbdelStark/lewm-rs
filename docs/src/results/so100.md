> **Motivation.** The SO-100 result extends LeWM to a 6-DOF real-robot
> dataset. The run converges in under 15 minutes on A10G-large with the
> same training stack as PushT.

The trajectory matches PushT qualitatively:

- **Steps 1–500.** SIGReg-dominated; encoder distribution shaping.
- **Steps 500–2500.** SIGReg falls by 3 orders of magnitude.
- **Steps 2500–5000.** Cosine-tail refinement.

The 5 k-step budget is enough to converge well below the
TOL-007 / TOL-008 / TOL-009 collapse thresholds while staying within
the 15-minute wall-time target.

## 3. Differences from PushT

| Aspect | PushT | SO-100 |
|--------|-------|--------|
| Action dim (raw) | 2 (x, y) | 6 (joint targets) |
| Steps | 50 000 | 5 000 |
| Wall time | 318 min | 14 min |
| Weight decay | 0.05 | 0.01 |
| Warmup | 1 000 steps | 500 steps |
| Action smoother input ch | 2 | 6 |
| Dataset size | 2 092 476 windows | 6 559 frames / 50 episodes |
| Eval | success rate (simulator) | latent-MSE (open loop) |

The most material differences are the action dim (smoother input
channels: 2 → 6) and the dataset scale (~300× more windows in PushT).
The smaller SO-100 dataset is the reason the run converges in 5 000
steps rather than 50 000 — there is less data to fit and a smaller
training budget suffices.

## 4. Notes

### 4.1 Same training stack

The training stack is identical to PushT's: `lewm-train`, the same
trainer state machine, the same AdamW + cosine schedule, the same
checkpoint protocol, the same parity contracts. The only changes are
the config (`configs/so100.toml`) and the dataset loader
(`So100Dataset` in place of `PushtDataset`).

This is the design payoff of the parity work in
[Part VI](../parity/why-parity.md): once the architecture is correct
and the parity tests pass, dropping in a new dataset takes one config
and one loader.

### 4.2 The held-out eval

The SO-100 training reserves episodes 45..50 for held-out evaluation.
Latent-rollout MSE on this split is the eval metric (see
[SO-100 eval](../planning/so100-eval.md)); the eval itself is pending.

### 4.3 Warm-start ablation

A warm-start training run (initialised from PushT step 50000) is
planned but not yet launched. The expected result, per LeWM's design,
is improved latent-MSE on the SO-100 eval split — see
[Warm-start ablation](../planning/warm-start.md).

## 6. Reproducibility

```sh
# Local short run
cargo run -p lewm-train -- \
    --config configs/so100.toml \
    --device cpu \
    --output-dir /tmp/lewm-so100 \
    --max-steps 10 train

# Cloud full run
scripts/launch_hf_job.py jobs/full_so100.yaml
```

With seed = 0, the run is reproducible to within TOL-005.
